# 已完成功能列表

## CP0 阶段进展概览
- **工程脚手架**：完成 Android Compose 工程初始化，配置根、模块级 `build.gradle.kts`，启用 Material 3 主题与基础资源。
- **主界面骨架**：按照剪映三栏布局搭建主页，包含组件库面板、手机化画布预览与属性面板，并提供顶部保存/预览入口及底部工具栏。
- **画布表现**：画布以手机框架呈现，强调“预览即进入应用”，避免依赖真机参数，保障后续代码生成的尺寸兼容性；新增状态栏/画布/底部栏三段式假手机视图，对齐上中下布局示意。
- **交互联动**：组件库支持选中高亮，画布预览与属性面板实时反映当前组件与缩放比例，提供滑杆调整 50%~250% 缩放与真机等效提示。
- **区域化画布**：画布拆分为顶部/中部/底部三段式假手机区块，可在属性面板切换组件所属区域，预览视图同步显示区块数量与状态。
- **属性编辑**：为画布图层增加宽度/高度/透明度滑杆与区域选择器，锁定状态下阻止修改，确保“画布即手机”视图与属性编辑保持一致。
- **对齐与留白**：新增组件水平对齐（左/居中/右）和内边距滑杆，宽度滑杆随内边距自动收敛，并在假手机画布中按对齐落位与留白展示。
- **手机端尺寸约束**：设备预设新增 720p/1080p/2K 的宽高参数，属性面板宽高滑杆随手机尺寸与画布分区自动限值，突出“仅按假手机尺寸计算”。
- **假手机布局校验**：属性面板新增“假手机校验”卡片，依据设备尺寸与区域高度提示宽高越界或透明度过低的风险，引导调整到手机端可用范围。
- **图层与画布同步**：可将组件一键添加到画布，中央画布与右侧属性面板展示图层列表，支持聚焦、显示/隐藏与锁定/解锁，上下移动排序、复制与删除，保持“预览即真机”的同步体验。
- **多设备假屏支持**：提供 720p/1080p/2K 等多机型尺寸选项，组件属性仅基于假手机计算，预览按钮可在编辑/预览模式间切换，模拟直接进入应用。
- **打包支持**：统一入口为 `run.sh`，提供依赖刷新、环境检查、完整/增量构建（含自动安装）以及独立安装器触发能力，在 Termux 环境可直接 `bash run.sh [选项]` 完成日常打包与安装；构建前锁定 SDK 下载架构为 linux/arm64，强制使用项目 Wrapper 并清理隔离 `.gradle-mobile` 缓存与 PC 架构 aapt2，若缺少本机 ARM64 aapt2 则自动尝试安装并强制 override 确保编译稳定。
- **文档体系**：`README.md`、《Termux 打包说明》《构建与测试记录》《开发更新记录》均已同步描述当前功能与打包使用方式，确保全中文文档可查。
